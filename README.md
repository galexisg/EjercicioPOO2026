# Evaluación Práctica: Pilares de la Programación Orientada a Objetos en Java

## 📌 Contexto
Escuela Superior Franciscana Especializada / Ágape  
Técnico en Ingeniería de Desarrollo de Software  
Módulo: Creación de Aplicaciones en Java SE  

---

## 🎯 Objetivo
Desarrollar ejercicios que evidencien la aplicación de los **pilares de la Programación Orientada a Objetos (POO)** en Java:  
- **Abstracción**  
- **Encapsulamiento**  
- **Herencia**  
- **Polimorfismo**

---

## 📝 Instrucciones
1. Examine las tres clases presentadas en la guía y **extraiga los elementos comunes** que puedan declararse en una **superclase**.  
2. Desarrolle la estructura de clases implementando los pilares de la POO en Java.  
3. En la clase **Padre**, cree un método que deberá ser **sobrescrito en las clases derivadas** (ejemplo de polimorfismo por herencia).  
4. Cada **pareja de estudiantes** deberá crear una **rama en el repositorio** para registrar su avance:  
   - `main` → versión estable.  
   - `develop` → rama de integración.  
   - `NombreApellido1-NombreApellido2`. → ramas de cada pareja de estudiantes.  
5. Deje constancia de los códigos desarrollados en un **proyecto de tipo Aplicación Java**.  
6. Documente el proceso en este repositorio, incluyendo:  
   - Código fuente en carpetas organizadas.  
   - Capturas de pantalla de la ejecución.  
   - Explicación breve de cómo se aplican los pilares en cada clase.  

---

## ✅ Entregables
- Proyecto Java con las clases implementadas.  
- Repositorio en GitHub con ramas (`NombreApellido1-NombreApellido2`, ramas de cada pareja).  
- Documento de evidencias (`docs/evidencias.md`) con explicación y capturas.  
- Este archivo **README.md** como guía principal del repositorio.  

---

## 🚀 Notas
- Recuerda aplicar **buenas prácticas de programación**: nombres claros, comentarios breves y consistencia en la estructura del código.  
- El polimorfismo debe demostrarse con la sobrescritura del método de la clase **Padre** en cada clase derivada.  
- El historial de commits y Pull Requests será parte de la evidencia de colaboración.
